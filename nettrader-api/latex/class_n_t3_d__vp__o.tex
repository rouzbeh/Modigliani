\subsubsection{Detailed Description}
\hyperlink{class_n_t3_d__drv__vp__o}{NT3D\_\-drv\_\-vp\_\-o} basically provides an implementation for all the functionality or infrastructure defined by this class, which is assigned to an \hyperlink{class_n_t3_d__abstract__vp__o}{NT3D\_\-abstract\_\-vp\_\-o} which uses this infrastructure. In this context it is possible that an \hyperlink{class_n_t3_d__abstract__vp__o}{NT3D\_\-abstract\_\-vp\_\-o} can be assigned to different drivers. On the other side the abstract viewport should provide all drawing within the \hyperlink{class_n_t3_d__o_a09f0d769add7e30f534dd792371b444c}{Draw()} method for the viewport drivers, which is the only method called from the driver classes. In the other direction the abstract classes should actually avoid to call the \hyperlink{class_n_t3_d__o_a09f0d769add7e30f534dd792371b444c}{Draw()} method themselves. \begin{Desc}
